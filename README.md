  В рамках работы в компании моими задачами являлись:

    1. Сопровождение сайта https://khmarketing.ru/ (Функционал: основная информация о компании, просмотр и отклик на вакансии, загрузка отчётов по выкладке товара мерчандайзерами, просмотр отчётов компаниями заказчиками/поставщиками);

    3. Сопровождение мобильного приложения https://play.google.com/store/apps/details?id=ru.khmarketing.app (Функционал: общий чат всех мерчандайзеров, загрузка отчётов и получение обратной связи, выходы на точку с проверкой геопозиции);
    4. Сопровождение настольного приложения менеджеров компании (Функционал крайне большой, основные разделы: Договора, Товары, География, Заработная плата, Персонал, Мерчандайзинг (Проверка отчётов от мерчандайзеров и работа с ними), Супервайзинг (аудиты), Отчёты (аналитические, не связанные с отчётами по выкладке товаров мерчандайзерами));
    5. Сопровождение связанной серверной части (подробнее в пункте Архитектура + технологии).
    
    Мои достижения:
    - Тотальная автоматизация процесса синхронизации между локальной и внешней базами данных. Ежедневно на исправление ошибок в ходе этого процесса я мог тратить до 2-3 часов, работа менеджеров стопорилась. Сейчас же все ошибки исправляются автоматически;
    - Оптимизация существующих SQL-запросов - внедрил логику отката транзакций, логирования и обработку ошибок в важнейших запросах;
    - Создал приложение "Менеджер проброса периодов": у всех входящих от мерчандайзеров отчётов и проектов есть свои периоды, которые нужно выставлять им в зависимости от их формата отчётности (недельные, двухнедельные, ежемесячные). Ранее было необходимо после рабочего дня проходится по каждому проекту вручную и устанавливать этот отчётный период, что занимало до 10 минут на 1 проект (из-за большого количества торговых точек в каждом проекте) + сильно нагружало БД. В итоге раз в неделю приходилось тратить до 6 часов времени на проброс этих периодов в течение двух дней. Моё приложение автоматически устанавливает отчётные периоды по каждому проекту и каждой точке на месяц вперёд, причем делает это без какого-либо вмешательства от пользователя и за счёт высокой оптимизации запросов минимально нагружает базу данных.
    - Десятки доработок настольного приложения менеджеров: реализация множественной быстрой работы там, где ранее операции делались долго и кропотливо по 1 запросу; изменения полной логики финансовой части всей системы из-за ввода нового закона о НДС, включая выгружаемой отчётности и математических формул; добавление новых фильтров, полей, обработчиков, кнопок по всему приложению; интеграция Яндекс.Геокодер API в приложение для получения коордиат точек по строке адреса и так далее;
    - Создание с нуля карты вакансий компании с использованием Яндекс.Карт: https://khmarketing.ru/vacanciesMap/map.html
    - Обновление кода веб-приложения под новые версии и стандарты php и Vue.js (за счёт этого ускорение работы сайта).
    
    Архитектура + технологии, со всем этим я успел поработать не единожды:
    1. Сайт - Vue.js, php;
    2. Мобильное приложение - Java, Kotlin;
    3. "Внешняя" база данных сайта и мобильного приложения - PostgreSQL;
    4. REST API для связи сайта, мобилки, внешней БД и локальной БД офиса - php, laravel, Yii 2;
    5. "Внешний" сервер для сайта и мобильного приложения, на котором хостится API и сайт - Linux Ubuntu, Nginx, Apache, Far Manager, ISP Manager;
    6. Настольное приложение менеджеров - C#, WinForms;
    7. Телеграм бот - Python, aiogram;
    8. "Локальная" база данных для бота и настольного приложения + хранения абсолютно всех данных, в том числе с внешней базы данных - SQL Server, много хранимых процедур и функций на T-SQL;
    9. "Локальный" (внутриофисный) сервер, на котором хостится "локальная" БД - Windows Server;
    10. Проект синхронизации для выгрузки данных с локальной БД во внешнию и наоборот - C#;
